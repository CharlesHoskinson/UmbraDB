UmbraDB · Release program

The road to 1.0.0 ← 0.1.0

A local, single-writer, PostgreSQL-backed temporal & content-addressed store for Midnight. Twenty gate items across five OpenSpec changes. The one that matters: what stands between here and 1.0.0 isn’t a feature — it’s a frozen public surface to version.

Scope 5 primitives + wallet envelope Gate items 20 OpenSpec changes 5 Non-goals not an ORM · not distributed
01

The thesis

Keystone

Freeze an API, not ship a feature

The package is private:true with no exports/types and no index.ts — nothing is versionable yet. 1.0.0 = the five primitives behind a frozen, documented surface.

One real blocker

The cursor can outrun durable data

CheckpointStore.save() can’t join a caller transaction, so a watermark can commit ahead of the checkpoint it points at → a silent gap on crash-resume. Everything else is P1 or docs.

Security posture

No code blocker under the trust model

Single trusted writer holds. The InfoSec gate is a threat-model doc + a supply-chain CI gate. The cross-wallet dedup “oracle” is a documentation obligation; keyed chunking is 1.1.

Critical path

Order is set by dependencies, not priority. The keystone signature change comes first; the test suite is the long pole; the API freeze waits until save() is final.

06

What’s deliberately not a blocker

These look like P0s but aren’t. Called out so they don’t creep back onto the blocker list.

save() idempotency  P0 → P1 / doc

A lost-ack retry produces a benign identical-content duplicate under load-latest semantics — not corruption. Its real deadline is “with Sprint 9”; grouping it with the genuinely silent-divergence G5 dilutes P0.

Perf hot-path fixes  P0 → baseline-only

Batching and N+1 fixes matter, but no perf number gates 1.0 — only that a baseline exists. The batching must merely precede the baseline it’s measured against.

Dedup oracle · plaintext keys  High → docs obligation

“High” is the severity of an excluded deployment. Under the single-trusted-writer model both channels require being the writer; the 1.0 obligation is documentation, not code. Fixed-size 4 MiB chunking (not content-defined) narrows it further.
